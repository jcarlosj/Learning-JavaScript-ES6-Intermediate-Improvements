Promises

  • EL flujo de información de internet tiene característivas asíncronas, lo que significa que mientras esperamos el resultado de una operación como por ejemplo que carguen los datos de una página web, un programa puede realizar otras operaciones y utilizarlo cuando el resultado esté listo
  • Las "promesas" (promises) son objetos que representa esta clase de operaciones y los datos que se obtienen
  • Las promesas nos sirven para administrar las funciones de callback, procesos asíncronos. Un callback es una función llamada desde otra función, a la multiplicidad de callbacks es lo que se conoce como el "hell the callbacks" (Infinidad de llamados) Las promesas mejoran la situación
  • Una promesa tiene tres estados: pending, fulfilled (success) y rejected (error)
  • Las promesas se pueden encadenar
  • Para enviar la información dentro de las promesas, se envían dentro de resolve() y de reject(), y dentro de return
  • Con throw cortamos las promesas
  • Las promesas están basadas en un standar que se llama Promises a+
  • Existen muchos paquetes que nos permiten antes de ES6 hacer uso de las Promesas
    • Accius
    • Angular tiene un manejador de Promesas toPromise
    • jQuery tiene deferred
    • Polyfill es6 promise http://github.com/stefanpenner/es6-promise
    • awai
    • http://polyfill.io/v2/docs

Crear promesas

  • new Promise ( function( resolve, reject ) {});
    tenemos una función anónima (se puede hacer con Fat Arrows)
  • La opción resolve() regresa el valor correcto (success)
  • La opción reject() regresa el valor erróneo (error)

Llamar a las promesas

  • Tiene un método que es "then", lee el valor correcto (success)
  • Tiene un método que es "catch", lee el valor erróneo (error)

Más de una promesa

  • Encadenar varias promesas (Cuando unos datos dependen de otros)
  • all[] indica si cumple todas las promesas
  • race[] indica si cumple "alguna" de las promesas
